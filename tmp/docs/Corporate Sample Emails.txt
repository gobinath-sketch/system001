Topic 1:
Airflow Fundamentals & Architecture
Subtopics:
Why move from TIDAL → Airflow (limitations vs benefits)
Explains the limitations of TIDAL in modern data ecosystems and how Apache Airflow provides better scalability, visibility, extensibility, and cloud-native orchestration.
Webserver
Provides the Airflow UI used by support teams to monitor DAG runs, task status, logs, retries, and failures.
Scheduler
Responsible for scheduling DAGs, triggering task instances, and ensuring workflows run according to defined intervals and dependencies.
Metadata Database
Stores all Airflow state information such as DAG runs, task instances, execution history, and variables.
Executor
Defines how tasks are executed (local, distributed, Kubernetes, etc.) and controls task parallelism.
Workers
Actual nodes/processes where Airflow tasks run and perform job execution.
DAG, Task, Operator, Hook, Connection
Introduces core Airflow concepts used to define workflows, individual job steps, integrations, and external system connectivity.
Airflow UI walkthrough
Demonstrates how to navigate DAG views, task logs, execution timelines, and error details from a support perspective.
Hands-On
Access Airflow UI
Log in to the Airflow web interface and become familiar with dashboards and navigation.
Explore sample DAGs
Review existing DAGs to understand structure, dependencies, and scheduling behavior.
Identify task states & logs
Learn how to interpret task states such as success, failed, retry, and running, and analyze logs for troubleshooting.
Topic 2:
DAGs & Scheduling (TIDAL Mapping)
Subtopics:
DAG structure & scheduling concepts
Explains how workflows are defined as DAGs and how scheduling drives execution timing.
CRON vs Airflow schedule_interval
Compares traditional CRON expressions with Airflow’s scheduling and execution-date-based model.
Task dependencies (parallel vs sequential)
Shows how tasks can run sequentially or in parallel based on dependency definitions.
Retries, SLA, timeout
Covers fault tolerance features used to manage failures, performance expectations, and delayed tasks.
Backfill & catchup
Explains how Airflow handles missed schedules and historical data reprocessing.
TIDAL → Airflow Mapping
Job → Task
A single executable unit in TIDAL maps to an individual task in Airflow.
Job Group → DAG
A collection of related TIDAL jobs is represented as one Airflow DAG.
Calendars → schedule_interval
TIDAL calendars are replaced by Airflow scheduling expressions.
Job dependencies → task dependencies
Upstream and downstream relationships are explicitly defined in DAG logic.
Alerts → callbacks / email / Slack
Notification handling moves to Airflow callbacks and integrations.Hands-On
Create a simple DAG
Build a basic Airflow DAG with multiple tasks and dependencies.
Convert a TIDAL workflow into an Airflow DAG (conceptual)
Understand how an existing TIDAL job flow would be represented in Airflow.
